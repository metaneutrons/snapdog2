Deploy Falcon with your application
How to deploy Falcon SDK together with your application ### Procedure title 1. If you are deploying for the .NET Framework the following software must be installed on the target system: - .NET Framework 4.8 (or higher) When deploying as a .NET 6 (or higher) Application you have the option to deploy as a framework dependent application (FDD) or as a self-contained application (SCD). For FDD application the .NET runtime must be installed on the target system. An SCD application does not need any framework to be installed. For more Information see ![External Link](../media/ExternalLink.png)Microsoft documentation 2. The following Falcon SDK files must be installed on the target system in the application directory: - Knx.Falcon.Sdk.dll - Knx.Falcon.dll - Knx.Falcon.ApplicationData.dll (if used) - knx\_interfaces.xml (if USB shall be used) - knx\_master.xml in schema version 2.3 (optional) In addition, the following dependencies might be required: - Bouncy Castle 1.9.0 (required only if KNX Point API is used) - log4net 2.0.12 (required only if [logging](e73c9fc8-1834-4088-80a5-1d8e41f7fcaf.htm) is configured to use log4net) - Microsoft.Bcl.AsyncInterfaces 7.0.0 (required only for .NET Framework) - MeaMod.DNS.dll - a signed version of MeaMod.DNS 1.0.70 (required only if KNX Point API is used) - Microsoft.Win32.Registry 5.0.0 (required only for .NET 6 or higher on Windows) - Newtonsoft.Json 13.0.3 (required only if KNX Point API is used) - System.Formats.Cbor 7.0.0 (required only if KNX Point API is used) - System.Memory 4.5.5 - System.Net.Http 4.3.4 (required only if KNX Point API is used) - System.Runtime.CompilerServices.Unsafe 6.0.0 - System.Security.Cryptography.ProtectedData 7.0.0 - System.Security.Cryptography.Xml 7.0.0 - System.Threading.Channels 5.0.0 - System.Threading.Tasks.Dataflow 7.0.0 Copyright Â© 2010-2024 KNX Association cvba, Brussels, Belgium

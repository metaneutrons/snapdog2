Logging
Logging is an invaluable tool when it comes to diagnosing problems. Falcon logging can be adapted to integrate into any existing logging system already in place in the client application. Pre-defined logging systems #### No Logging In the rare case that logging is not wanted, it can be completely switched off by using the following code before accessing any other Falcon class: C# VB C++ F# Copy Logger.Factory = new NullLoggerFactory(); Logger.Factory = New NullLoggerFactory() No code example is currently available or this language may not be supported. No code example is currently available or this language may not be supported. #### log4net Logging ![External Link](../media/ExternalLink.png)log4net is a popular logging system which is highly configurable and provides output to a rich set of targets, including files, databases, and network protocols. To use log4net, you need to initialize and configure it in your application before Falcon is used. See the log4net documentation on how to do this. This is the default setting. Falcon uses log4net v2.0.8. Custom Logging Provider You may also provide your own logging facility: - Create a class implementing the [IFalconLogger](7e425923-e1cc-8dc4-632e-b44b6df61ef0.htm) interface. - Create a class implementing the [IFalconLoggerFactory](3f8b1fe4-f3e8-7798-0d3a-f702c52391b4.htm) interface. - Register an instance of the latter class as [Factory](eca21355-b095-72cc-deba-523a01d68f96.htm) before accessing any other Falcon class.

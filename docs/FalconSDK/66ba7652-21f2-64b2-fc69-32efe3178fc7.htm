IoTOperationPolicy
A policy that determines how a function is performed, and in particular what happens if the function can for some reason not be performed.

This entails specifying how many times the operation shall be attempted in total. this includes a NumberOfInnerAttempts that determines how many times supplementary operations shall be performed. e.g. operations such as determining the device IPv6 address and port given its serial Number.

Timeouts are also specified. Note that for some operations both minimal and maximal timeouts are necessary.

  • MaximumTimeout - Determines when to abort waiting to see if a operation was successful and treating the operation as a failure.
  • MinimumDuration - Determines for operations that are open ended for how long at the very least waiting for some events to occur shall be done.

IoTPublisher
Entry in the Publisher table of a KNX IoT device.

The publisher table fulfills the purpose of determining from which endpoints to receive group communication

The required Information is:

  • Id - ID of entry unique with respect to the table
  • GroupAddresses - Group Addresses for which to message the specified recipient

And either of these can then be used to uniquely identify the publisher endpoint:

  • The Url - Defines a url including IPv6 address and port to tell the device from which publishers to receive.
  • The IndividualAddress and InstallationId - Define the individual address and installation id of the target device which the device can use to determine IPv6 address and port.
IoTRecipient
Entry in the Recipient table of a KNX IoT device.

The recipient table fulfills the purpose of determining to which endpoints group messages shall be sent to.

The required Information is:

  • Id - ID of entry unique with respect to the table
  • GroupAddresses - Group Addresses for which to message the specified recipient

And either of these can then be used to uniquely identify the recipient endpoint:

  • The Url - Defines a url including IPv6 address and port to tell the device how to reach the recipient.
  • The IndividualAddress and InstallationId - Defines the individual address and installation id of the target device which the device can use to determine IPv6 address and port.

Some additional configuration is possible such as making messages non-confirmable (see NonConfirmable) or specifying the maximum amount of retransmissions (see MaximumRetransmissions) or changing the the path used for group communication to something other than .knx (see Path).

The following outlines the steps in which recipient tables are used:

  1. Checking the point value

    Check if one of the following situations have occurred:

    • A Point value has changed and the elapsed time since the last update is greater than the minimum repetition time.
    • The elapsed time since the last update is greater than the heartbeat

  2. Checking group object configuration flags

    The KNX IoT device checks the group object configuration flags to see if the respective Point value shall be published.

  3. Checking recipient table items

    The KNX IoT device searches group object table items and Functional Point recipient table items that have the same group address configured.

  4. Send group message to recipient

    The KNX IoT device sends a group message to the network.

  5. Update the last sent time stamp

    If the message was sent to the network, then the device updates the last sent time stamp.
    This step is only applicable if heartbeat or minimum repetition timeout can be configured.

IIoTDevice
Provides access to various device management actions for IoT devices. A device is uniquely identified by its serial number which is used to send requests to perform the desired actions.

The following facilities are provided:

Reading information regarding the device state:

  • Programming Mode - ReadProgrammingModeAsync(IoTOperationPolicy, IoTOperationToken)
  • Load State Machine State - ReadLoadStateMachineStateAsync(IoTOperationPolicy, IoTOperationToken)
  • Individual Address - ReadIndividualAddressAsync(IoTOperationPolicy, IoTOperationToken)
  • Installation Id - ReadInstallationIdAsync(IoTOperationPolicy, IoTOperationToken)
  • Group Object Table - ReadGroupObjectTableAsync(IoTOperationPolicy, IoTOperationToken)
  • Recipient Table - ReadRecipientTableAsync(IoTOperationPolicy, IoTOperationToken)
  • Publisher Table - ReadPublisherTableAsync(IoTOperationPolicy, IoTOperationToken)
  • Parameters - ReadParametersAsync(IoTOperationPolicy, IoTOperationToken)
  • Fingerprint - ReadFingerprintAsync(IoTOperationPolicy, IoTOperationToken)
  • Hardware type - ReadHardwareTypeAsync(IoTOperationPolicy, IoTOperationToken)

Writing configuration information to alter the device state:

  • Programming Mode - WriteProgrammingModeAsync(Boolean, IoTOperationPolicy, IoTOperationToken)
  • Load State Machine State - WriteLoadStateMachineStateAsync(IoTLoadStateMachineCommand, IoTOperationPolicy, IoTOperationToken)
  • Individual Address and Installation Id - WriteIndividualAddressAndDefaultInstallationIdAsync(IndividualAddress, IoTOperationPolicy, IoTOperationToken)
  • Group Object Table - WriteGroupObjectTableAsync(IEnumerable IoTGroupObject , IoTOperationPolicy, IoTOperationToken)
  • Recipient Table - WriteRecipientTableAsync(IEnumerable IoTRecipient , IoTOperationPolicy, IoTOperationToken)
  • Publisher Table - WritePublisherTableAsync(IEnumerable IoTPublisher , IoTOperationPolicy, IoTOperationToken)
  • Parameters - WriteParameterAsync(IoTParameter, IoTOperationPolicy, IoTOperationToken)

Performing actions on the device:

  • Restart - RestartAsync(IoTOperationPolicy, IoTOperationToken)
  • Reset - ResetAsync(ConfirmedRestartEraseCode, IoTOperationPolicy, IoTOperationToken)
IIoTGroupCommunication
Allows to send and receive KNX IoT S-Mode group messages.
IIoTNetwork
Provides access to KNX IoT network management procedures.

It provides the following network facilities:

  • General Scan - Find any device and get serial number or individual address e.g. DiscoverDeviceInfosOfAllDevicesAsync(IoTOperationPolicy, IoTOperationToken)
  • Filtered Scan - Find devices with some restriction e.g. ScanIndividualAddressRangeAsync(IndividualAddress, IndividualAddress, IProgress Int32 , IoTOperationToken)
  • Programming Mode Scan - Find all devices in Programming mode e.g. ReadDeviceInfoByProgrammingModeAsync(IoTOperationPolicy, IoTOperationToken)
  • Ping - Check if a device is reachable / individual address is occupied e.g. PingIndividualAddressAsync(IndividualAddress, IoTOperationPolicy, IoTOperationToken)
  • Conversion - Convert between Individual Address and Serial number e.g. ReadSerialNumberByIndividualAddressAsync(IndividualAddress, IoTOperationPolicy, IoTOperationToken)
IIoTSecurity
Provides access to functions relevant for managing the security of devices and onboarding new devices.

For the secure onboarding of new devices:

  • Tool Key Configuration - AuthenticateDeviceAsync(String, Func SecureKey, Task SecureKey , IoTOperationPolicy, IoTOperationToken)

For configuring and reading authentication tables:

  • Writing the authentication table - WriteAuthenticationTableAsync(IEnumerable IoTAccessToken , IoTOperationPolicy, IoTOperationToken)
  • Reading the authentication table - ReadAuthenticationTableAsync(IoTOperationPolicy, IoTOperationToken)>
  • Reading a single IoTAccessToken by its id - ReadAccessTokenItemAsync(String, IoTOperationPolicy, IoTOperationToken)
IIoTSoftwareUpdate
Provides software update functionality for a KNX IoT device.

This allows to upgrade the software on an IoT device in two ways. This may either be done by pushing a software update package directly to an IoT Device

or instructing the device to pull the specified software update package from an specified software update server using a specified protocol.

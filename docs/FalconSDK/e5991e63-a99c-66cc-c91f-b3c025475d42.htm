[Top](#PageHeader) Remarks The parameters are used in the following way to determine which USB connection shall be opened: - If [DevicePath](fefecde0-ced9-b326-6a15-10374f2fc4fa.htm) is set, it is first tried to open the USB interface with this USB device path. If this fails, the algorithm continues with the next step. - If [SerialNumber](3b1c2517-947a-0d6a-cf97-4ee17d9b6175.htm) is not [None](44f0792f-441d-4c8f-4ece-24cff24b35b2.htm), the connected USB interface with this KNX serial number is opened. If no USB interfaces with this KNX serial number is found, opening fails. - If [PreferredAddressToOpen](dafdde8f-a87f-4eb4-23df-1e9c15703f1e.htm) is set, the connected USB interface with this individual address is opened. If no or multiple USB interfaces with this individual address are found, opening fails. - The first available USB interface is opened. **Connection string support** Connection String Copy "Type=Usb;DevicePath=usbpath" Connection String Copy "Type=Usb;PreferredAddressToOpen=indaddr" e.g. "Type=USB;PreferredAddressToOpen=1.1.250" Connection String Copy "Type=Usb;SerialNumber=serno" e.g. "Type=USB;SerialNumber=00FA:01020304" Connection String Copy "Type=Usb;...;RFMulti=/" Connection String Copy "Type=Usb;...;AutoReconnect=/" If parts of the connection string are omitted, the default values of the corresponding properties are used. **Additional information** On Linux, the user running Falcon must be granted read and write access to the USB HID device, e.g. using Shell Copy sudo chmod a+rw /dev/hidraw0 See Also #### Reference [Knx.Falcon.Configuration Namespace](8057f884-553e-ad7e-77ee-8d8d0f07a343.htm) Copyright © 2010-2024 KNX Association cvba, Brussels, Belgium

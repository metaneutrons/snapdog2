IoTRecipient Class
Entry in the Recipient table of a KNX IoT device. The recipient table fulfills the purpose of determining to which endpoints group messages shall be sent to. The required Information is: - [Id](d3dfac19-8f6b-4069-f443-bd0d721e1cef.htm) - ID of entry unique with respect to the table - [GroupAddresses](7ee20d08-ce7d-8d18-e311-ab72517dde84.htm) - Group Addresses for which to message the specified recipient And either of these can then be used to uniquely identify the recipient endpoint: - The [Url](862bcc91-9121-383e-1153-18e29aec630d.htm) - Defines a url including IPv6 address and port to tell the device how to reach the recipient. - The [IndividualAddress](d58b7c76-d1b6-47d7-b6aa-37f86ba60ded.htm) and [InstallationId](93bb37ed-0303-8aa4-9cc7-dd51972f622e.htm) - Defines the individual address and installation id of the target device which the device can use to determine IPv6 address and port. Some additional configuration is possible such as making messages non-confirmable (see [NonConfirmable](d2c5fdf5-fc93-8157-8df6-c239655566da.htm)) or specifying the maximum amount of retransmissions (see [MaximumRetransmissions](3b9ccf00-264a-5a56-ab48-89b37b832cc0.htm)) or changing the the path used for group communication to something other than .knx (see [Path](e2ef41ff-91fd-3a85-352f-4f1579a951db.htm)). The following outlines the steps in which recipient tables are used: 1. **Checking the point value** Check if one of the following situations have occurred: - A Point value has changed **and** the elapsed time since the last update is greater than the minimum repetition time. - The elapsed time since the last update is greater than the heartbeat 2. **Checking group object configuration flags** The KNX IoT device checks the group object configuration flags to see if the respective Point value shall be published. 3. **Checking recipient table items** The KNX IoT device searches group object table items and Functional Point recipient table items that have the same group address configured. 4. **Send group message to recipient** The KNX IoT device sends a group message to the network. 5. **Update the last sent time stamp** If the message was sent to the network, then the device updates the last sent time stamp. *This step is only applicable if heartbeat or minimum repetition timeout can be configured.* Inheritance Hierarchy System Object   Knx.Falcon IoTRecipient **Namespace:**  [Knx.Falcon](66ba7652-21f2-64b2-fc69-32efe3178fc7.htm) **Assembly:**  Knx.Falcon (in Knx.Falcon.dll) Version: 6.3.0 \[2024-12-07|33a4f8ce8366fff4f9e3f5340a78dd8664732dba|origin/releases/6.3-profiterole\] Syntax C# VB C++ F# Copy public class IoTRecipient : IEquatable, IObjectRepresentationSerializable Public Class IoTRecipient Implements IEquatable(Of IoTRecipient), IObjectRepresentationSerializable public ref class IoTRecipient : IEquatable, IObjectRepresentationSerializable type IoTRecipient = class interface IEquatable interface IObjectRepresentationSerializable end The IoTRecipient type exposes the following members. MaximumRetransmissions
Defines the amount of allowed retransmissions for sending group messages.

If this property is omitted then the following values are used:

  • For not confirmed messages: 0 retransmissions
  • For confirmed messages: 4 retransmissions
NonConfirmable
Defines if group messages shall be confirmed.

Per default this is set to false indicating that group messages shall be confirmed.

If this is true then non-confirmable requests will be sent. Retransmissions will not be possible in this case. (see also: MaximumRetransmissions)
Path
The resource path of the recipient used for sending group messages.

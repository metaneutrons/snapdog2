Version 6.0.2/6.0,3
Version 6.0.2/6.0.3 Changes in this Release This is a bugfix release. Here is a quick overview: - Falcon KNX Stack - Changed Falcon TL implementation according to AN210 - Falcon KNX/IP Stack - DeviceManagement connection: Improved re-connect behaviour on device restart - Bus interfaces - USB: Always set PID\_COMM\_MODE after after reception of M\_Reset.ind - USB: Always set PID\_FILTERING\_MODE\_SELECT after switching to Link Layer - USB: recover after failed connection attempt with disconnected bus voltage - DatapointType converters - DPT 22 is now correctly classified as compound type. - Device Management - MaxApduLength was not set correctly for connection-less connection if the device does not support connection-less DeviceDescriptorRead Version 6.0.3 has no functional changes but corrects a problem that the classes IpDeviceDiscovery and UsbDeviceDiscovery were not accessible by Falcon clients.

Remarks If the discovered device does not support tunneling at all, the returned list is empty. Otherwise, the first entry in the list is an entry without [IndividualAddress](a193faf7-9cc9-ba31-d223-dee65722eede.htm); if this is used to connect, the IP device may return any of the available tunneling slots. If the device supports Tunneling v2, in addition for each unique tunneling slot address, an additional entry is returned with [IndividualAddress](a193faf7-9cc9-ba31-d223-dee65722eede.htm) = slot address; if this is used to connect, exactly the requested slot is used.

ConfigurationFlags Enumeration
Information flags that determine which operations may be performed on a certain group object.   Member name Value Description
None 0 No operations can be performed on the group object
Read 8 Determines whether the group object may be read.

Depending on whether or not the flag is set the following behaviour is possible:

  • false = Group Object value can not be read.
  • true = When a read command to the Group Object occurs, the device responds with the value of this Group Object.
Write 16 Determines whether the group object may be written to.

Depending on whether or not the flag is set the following behaviour is possible:

  • false = Group Object value can not be written.
  • true = When a read command to the Group Object occurs, the device responds with the value of this Group Object.
Initialize 32 Determines whether the group object may be read after initialization.

Depending on whether or not the flag is set the following behaviour is possible:

  • false = Group Object value can not be read after initialization.
  • true = Group Object read request after initialization (e.g., power up) are permitted.
    To get a response the r-Flag on the producer side is expected to be r = true.
Transmit 64 Determines whether the group object may be transmitted.

Depending on whether or not the flag is set the following behaviour is possible:

  • false = Group Object value is not transmitted.
  • true = In the case of CoV, an event or heartbeat, the device sends its Group Object value.
    This is the default behaviour for outputs such as sensors (e.g. a push button).
Update 128 Determines whether the group object may updated.

Depending on whether or not the flag is set the following behaviour is possible:

  • false = Group Object value is not updated.
  • true = The Group Object value is updated with a group value response message if the flag Write = true.

STATUS_AUTHENTICATION_SUCCESS 0 This is returned as an OK result in response to a SESSION_AUTHENTICATE frame. The secure connection is established.
STATUS_AUTHENTICATION_FAILED 1 The is returned as an error result in response to a SESSION_AUTHENTICATE frame. Either the user is unknown or invalid or the password is invalid. The secure session is not established.
STATUS_ERROR_UNAUTHORIZED 2 TBD.
STATUS_TIMEOUT 3 This is sent bei the client or by the server to notify the peer that a timeout of any kind has occurred. The secure session is not established or is closed. For example this is sent by the server to the client when the server did not receive a SESSION_AUTHENTICATE frame after having sent a SESSION_RESPONSE frame within 10 s.
STATUS_KEEPALIVE 4 Sent by client to server to prevent the session from timing out when there is no other SECURE_WRAPPER traffic.
STATUS_CLOSE 5 Sent from the client to the server or from the server to the client to close the session immediately. Servers should respond to clients with a SESSION_CLOSE.

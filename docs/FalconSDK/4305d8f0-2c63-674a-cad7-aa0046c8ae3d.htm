[Top](#PageHeader) Remarks The parameters are used in the following way to determine which Tunneling Server shall be used: - If [HostAddress](be75f761-4534-fc02-eae0-10a8c2c7cd6d.htm) is set, it is first tried to contact the tunneling server with this host address. If this fails, the algorithm continues with the next step. - If [SerialNumber](d0444f88-ffde-0566-cc90-ab626c32526e.htm) is not equal to [None](44f0792f-441d-4c8f-4ece-24cff24b35b2.htm), the tunneling server is searched using standard KNXnet/IP discovery. - If there is only one device supporting Tunneling can be found, use it. See [IndividualAddress](a193faf7-9cc9-ba31-d223-dee65722eede.htm) on the further algorithm to determine the tunneling slot to use. **Connection string support** Connection String Copy "Type=IpTunneling;HostAddress=ipaddr;IpPort=port;ProtocolType=protocol;UseNat=True" With Tunneling v2, a specific tunneling slot can be requested: Connection String Copy "Type=IpTunneling;...;IndividualAddress=indaddr" Connection String Copy "Type=IpTunneling;...;AutoReconnect=/" If parts of the connection string are omitted, the default values of the corresponding properties are used.

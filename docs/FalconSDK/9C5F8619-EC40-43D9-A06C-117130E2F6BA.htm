KNX Falcon SDK

Version 6.3.0
Version 6.3.0 Changes in this Release (Possibly) breaking changes: - The Falcon exception classes do not support binary serialization any more as this is considered obsolete. See BinaryFormatter Obsoletion Strategy - The default [Factory](eca21355-b095-72cc-deba-523a01d68f96.htm) is now set to the Log4net logger only if all the following applies: - the file log4net.dll is present in the application base directory - the .NET system is not based on Mono - either the log4net assembly already is loaded in the process or the .NET system is .NET Framework 4.8 If this does not apply the [NullLoggerFactory](2de9d6b2-6fc3-70a5-d05f-d3edec6c9482.htm) is set instead. This avoids problems on systems where log4net may not work properly (as has been reported for Mono or MAUI on Android). Of course, you can still set [Factory](eca21355-b095-72cc-deba-523a01d68f96.htm) to Log4NetLoggerFactory or a user-supplied logger factory manually.

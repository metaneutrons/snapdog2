IoTOperationPolicy Properties
The [IoTOperationPolicy](f88614d5-4a6b-ace3-0b47-ac466d123ebf.htm) type exposes the following members. Properties
  Name Description
ActiveWaitingMaximumTimeout
The time after which the active waiting is aborted. During this time period every ActiveWaitingRefreshDelay the current state is polled.
ActiveWaitingRefreshDelay
This time is the interval between retries when actively waiting e.g. for a state change.
MaximumTimeout
The given time after which any individual operation is canceled. Take note that some operations consist of multiple steps so this timeout will be used for each step.
MinimumDuration
This timespan takes effect on operations such as discovery that wait for results to arrive for a set timespan.
NumberOfAttempts
The number of attempts determines the total number of attempts performed. This number shall always be at least 1.
NumberOfInnerAttempts
The number of attempts determines the number of attempts performed for operations that constitutes a partial step within the completion of another operation. An example would be an operation for reading a group objectTable, this value then determines how many attempts may be performed for reading each group value. This number shall always be at least 1.

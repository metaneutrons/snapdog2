IIoTNetwork Interface
Provides access to KNX IoT network management procedures. It provides the following network facilities: - General Scan - Find any device and get serial number or individual address e.g. [DiscoverDeviceInfosOfAllDevicesAsync(IoTOperationPolicy, IoTOperationToken)](5a88917f-cae4-0c3a-6548-8f3a375b1782.htm) - Filtered Scan - Find devices with some restriction e.g. [ScanIndividualAddressRangeAsync(IndividualAddress, IndividualAddress, IProgress](31fd2ace-67dd-8645-8da8-72b648c394b1.htm) Int32 , IoTOperationToken) - Programming Mode Scan - Find all devices in Programming mode e.g. [ReadDeviceInfoByProgrammingModeAsync(IoTOperationPolicy, IoTOperationToken)](24069d70-27b4-702e-3257-7a1afe221b23.htm) - Ping - Check if a device is reachable / individual address is occupied e.g. [PingIndividualAddressAsync(IndividualAddress, IoTOperationPolicy, IoTOperationToken)](e143433d-deba-33c1-22bf-720879c6aa80.htm) - Conversion - Convert between Individual Address and Serial number e.g. [ReadSerialNumberByIndividualAddressAsync(IndividualAddress, IoTOperationPolicy, IoTOperationToken)](d54ba913-94ef-3071-f0b4-e375b579bf96.htm) **Namespace:**  [Knx.Falcon](66ba7652-21f2-64b2-fc69-32efe3178fc7.htm) **Assembly:**  Knx.Falcon (in Knx.Falcon.dll) Version: 6.3.0 \[2024-12-07|33a4f8ce8366fff4f9e3f5340a78dd8664732dba|origin/releases/6.3-profiterole\] Syntax C# VB C++ F# Copy public interface IIoTNetwork : IDisposable, IAsyncDisposable Public Interface IIoTNetwork Inherits IDisposable, IAsyncDisposable public interface class IIoTNetwork : IDisposable, IAsyncDisposable type IIoTNetwork = interface interface IDisposable interface IAsyncDisposable end The IIoTNetwork type exposes the following members. [Top](#PageHeader) Remarks The class IIoTNetworkExtensions provides overloads for the functions of this interface that do not require an [IoTOperationPolicy](f88614d5-4a6b-ace3-0b47-ac466d123ebf.htm) to be specified and use a default policy instead that is appropriate for the respective actions. If a custom policy is provided by the user care should be taken since too short timeouts may cause failures and some functions are not suitable for retrying. Note that the network object should be disposed as soon as it is no longer needed.

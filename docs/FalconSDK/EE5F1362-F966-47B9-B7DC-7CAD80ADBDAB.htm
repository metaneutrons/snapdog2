KNX Falcon SDK

Version 6.1.1
Version 6.1.1 Changes in this Release The main purpose of this release is to support RF Multi communication. Besides this, it contains some improvements and bug fixes. Here is a quick overview: - Environment - ADD: Falcon is now compatible with the .NET Core globalization invariant mode. - API - ADD: [MinTimeBetweenSend](dce26676-53c0-63e9-0676-6a237d1d47da.htm) can be used to throttle bus communication. - Falcon KNX Stack - ADD: The RF Multi implementation has been completed in this release. - FIX: Gracefully handle interfaces reporting incoming messages that are not directed to the interface's individual address. - FIX: Re-connect a broken USB connection now works reliably. - Falcon KNXnet/IP Stack - FIX: Fixed possible Denial-of-Service attack via manipulated SEARCH\_RESPONSE. - FIX: Connection Channel ID 0 is now handled correctly - KNX Data Security - FIX: Improved interpretation of received sequence numbers, earlier Falcon versions could incorrectly suspect a replay attack in specific scenarios. - FIX: Ensure that two Sync requests are always separated by at least one second.

How to use Falcon from unmanaged code General information Falcon is a .NET class library which can be used "out-of-the-box" by .NET applications. Using standard .NET interop techniques, it is possible also for unmanaged (i.e. non-.NET) applications to access the required Falcon functionality. KNX does not provide such an API, but this section will give some hints how to create it. If this is not possible, you could wrap Falcon in a separate executable and perform any form of inter-process communication (IPC) to couple your application with this wrapper. An example is available from KNX support. COM clients If the client is able to use COM objects, a ![External Link](../media/ExternalLink.png)"COM callable wrapper" has to be implemented. See the linked Microsoft documentation for details. Create a .NET class library containing the classes to be used from the COM client. See ![External Link](../media/ExternalLink.png)"Exposing .NET Framework Components to COM" on how to do this. Implement the functionality by calling the Falcon SDK methods. You will need to either register the COM classes in the Windows registry using the Regasm.exe tool or use a technique called "registration-free COM".
